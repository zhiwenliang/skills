# Explanation Principles

Use this reference when designing or reviewing an explanatory article. Keep these ideas agent-facing; do not leak pedagogy terms into reader prose unless the article is about learning science.

## Reader Model First

An explanation works against a reader's current model. Declare:

- what the reader probably already knows;
- what confusion or wrong model the article repairs;
- what the reader should be able to do after reading.

Avoid serving beginners and experts with the same path. Beginners need more worked examples and fewer terms at once. Experts need sharper distinctions, edge cases, and tradeoffs.

## Causal Structure Over Labels

A label is not an explanation. For each major term, answer:

- What parts does it contain?
- What changes what?
- What happens first, second, and later?
- What tradeoff or constraint makes this design necessary?
- Where does the explanation stop working?

Good article thesis: "X works by doing Y, which solves Z but creates cost W."

## Depth Probes

Add a visible depth section for normal-length articles. It should include at least three of these probes:

- **Mechanism chain**: the sequence from cause to effect, with the intermediate step that usually gets skipped.
- **Constraint or tradeoff**: the pressure that makes the design or phenomenon necessary, and the cost it creates.
- **Counterfactual**: what would break or change if the mechanism were removed.
- **Model comparison**: why a tempting simpler model is incomplete.
- **Source-backed detail**: a specific detail from reliable sources that changes the explanation, not a decorative citation. Include this probe in every normal-length researched article.

This section prevents "definition plus example" articles from passing as deep explanations.

Each probe must be a real explanatory bullet, not a heading label. A weak probe says "There is a tradeoff." A strong probe names the pressure, the choice it forces, and the cost created by that choice.

## Cognitive Load Control

Readers lose the model when too many new elements arrive at once.

Operational rules:

- Introduce one new distinction at a time.
- Put definitions near first use.
- Use short sections with informative headings.
- Remove tangents that do not change the reader's model.
- Prefer diagrams when relation, flow, hierarchy, state, or contrast matters.

## Examples And Contrasts

Use examples to build boundaries.

- Worked example: shows how the mechanism operates.
- Contrasting case: shows what changes when one condition changes.
- Non-example: shows what the concept is not.
- Concrete-to-abstract progression: starts with a familiar case, then extracts the general model.

For novices, place worked examples before open prompts. For stronger readers, move faster to tradeoffs and discriminating cases.

## Analogies

Use analogies only when they map structure.

Checklist:

- Name the familiar source domain.
- State the target concept.
- Map the shared relations explicitly.
- State where the analogy breaks.
- Stop using the analogy once the real mechanism is clear.

Bad analogy: decorative comparison. Good analogy: a temporary bridge to the causal model.

## Misconception Repair

When a topic has common wrong models, repair them explicitly:

1. State the tempting wrong model.
2. Show the observation it cannot explain.
3. Introduce the better model.
4. Show how the better model handles both the easy case and the confusing case.

## Retrieval And Transfer

End with questions that force reconstruction:

- Prediction: "What happens if X changes?"
- Contrast: "Why is A not B?"
- Boundary: "Where would this explanation fail?"
- Paraphrase: "Explain this in your own words without using term X."

Avoid questions that only ask the reader to copy a phrase from the article.

At least two understanding checks should require transfer: prediction, contrast, boundary judgment, or "what breaks if..." reasoning. "What is X?" questions are allowed only as secondary warmups.

## Source Discipline

Use official, primary, or high-trust sources for factual claims. For normal-length researched articles, use at least two sources unless the user provides a single authoritative source. For current or disputed topics, date the framing. If source verification was not performed because the user asked for a quick draft, say so in the sources section instead of implying research.
